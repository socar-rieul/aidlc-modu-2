# Personas (테이블오더 서비스 — v2)

> **v2 (BYOD + QR 모델) 기준** — v1(공용 태블릿) 버전은 git tag `v1-shared-tablet`에 보존. 페르소나 권한·정체성은 v1과 동일하지만 **디바이스 = 본인 모바일 폰**으로 갱신, 페인 포인트·voice 보강.

총 **4 페르소나** — 매장 고객 2(P1 진우 30대 / P4 정희 50대 디지털 리터러시 하) + 매장 운영자 2(P2 점주 수민 / P3 알바 도윤).

## P1 — 매장 고객 "진우"

| 항목 | 내용 |
|------|------|
| 이름 | 진우 (30대 직장인) |
| 역할 | 매장 테이블에 앉은 손님 — 주문 주체 (v2: 일행과 함께 공동 장바구니 사용) |
| 사용 디바이스 | **본인 모바일 폰** (iOS Safari / Android Chrome). 테이블 QR을 자기 폰으로 스캔해 입장 |
| 사용 환경 | 매장 내부, 점심·저녁 피크 시간, 일행 1~3명. 일행 각자 자기 폰으로 동시 접속 가능 |
| 핵심 동기 | 직원을 부르지 않고 **즉시·정확하게** 메뉴를 보고 주문하고 싶다. 일행과 같은 화면을 보면서 메뉴를 의논하고 한 번에 시키고 싶다 |
| 니즈 | ① QR 한 번에 입장(앱 설치 X) ② 메뉴 사진·가격·설명 한눈에 ③ **일행과 공동 장바구니로 의논** ④ 누가 시키든 합산해서 한 번에 확정 ⑤ 테이블 전체 주문 내역으로 진행 상황 확인 |
| 페인 포인트 | 앱 설치 강요, 일행마다 따로 시키느라 메뉴 중복·누락, 본인 폰 화면 비좁아 작은 버튼 누르기 어려움 |
| 디지털 친화도 | 중~상 — 스마트폰 일상 사용, QR 스캔·카메라 익숙, 처음 보는 UI도 30초면 익힘 |
| 보안 민감도 | 낮음 (결제 없음, 개인정보 입력 없음) — 단 매장 외부에 주문이 새지 않는다는 신뢰는 필요 |
| Voice Quote | "QR 한 번 찍으면 같이 보면서 시킬 수 있는 거지? 일행 폰에서도 똑같이 보이고?" |

## P4 — 디지털 리터러시 부족 50대 고객 "정희"

| 항목 | 내용 |
|------|------|
| 이름 | 정희 (50대 후반 가족·지인 단위 외식 손님) |
| 역할 | 매장 테이블에 앉은 손님 — 주문 주체 (P1과 동일하나 디지털 리터러시 낮음) |
| 사용 디바이스 | **본인 모바일 폰** (자녀가 사준 스마트폰, 평소 카톡·전화 위주). 일행이 대신 QR 스캔해주는 경우도 흔함 |
| 사용 환경 | 가족·친구와 동석, 같은 테이블에 디지털 친화도 높은 일행(자녀·조카)이 있을 때가 많음 |
| 핵심 동기 | 직원을 부르고 싶지만 키오스크·QR 주문이 대세가 되면서 "내가 직접 잘 해보고 싶다" — **혼자서도 실수 없이** 주문 완료가 목표. 일행에게 매번 부탁하는 게 미안하다 |
| 니즈 | ① **QR 스캔이 한 번에 되는 단순한 안내** ② 글자·버튼이 큼지막해야 보임 ③ "지금 어디를 눌러야 하지?"가 한눈에 보이는 단순한 흐름 ④ 잘못 누른 것 같을 때 되돌릴 안전망 ⑤ 실수로 비싼 메뉴를 다량 시키지 않도록 **최종 확정 직전 한 번 더 확인** ⑥ 모를 때 도움을 주는 짧은 안내 ⑦ **본인 폰 시스템 폰트 크기 설정이 그대로 반영** |
| 페인 포인트 | **QR 스캔 자체가 첫 장벽**(카메라 앱 안 익숙), 본인 폰의 작은 화면, 작은 글자·아이콘만 있는 UI, 빠르게 사라지는 토스트, 다단계 메뉴 깊이, "장바구니 비우기" 같은 비유 용어, 모르고 누르면 자동 진행되는 버튼, **일행이 동시에 장바구니를 만지면 화면이 갑자기 바뀌어 당황** |
| 디지털 친화도 | **하** — 스마트폰은 카톡·전화·검색만 사용, 새 UI는 5~10분 학습 필요, QR 스캔은 가끔 해봤지만 자신 없음 |
| 보안 민감도 | 중 — 신용카드·이름이 안 들어간다는 점은 안심하지만 "실수로 비싼 거 시켜졌으면 어쩌지" 걱정 큼 |
| Voice Quote | "QR 어떻게 찍는 거야? 카메라만 갖다 대면 돼? 글자도 좀 키워봐, 안경 안 끼고도 보여야지. 누르기 전에 한 번만 더 물어봐 주면 좋겠어." |

## P2 — 점주 "수민"

| 항목 | 내용 |
|------|------|
| 이름 | 수민 (40대 자영업자) |
| 역할 | 매장 소유주 — 매장 인증 자격 보유, **모든 관리 기능 권한** |
| 사용 디바이스 | 매장 후방 데스크탑 / 카운터 노트북 (브라우저) — v2에서도 동일 (관리자 UI는 모바일 BYOD 영향 없음) |
| 사용 환경 | 영업 중 카운터에서 실시간 모니터링 + 영업 외 시간 메뉴·계정 관리 |
| 핵심 동기 | 직원·인건비를 줄이면서 **매장 흐름을 한눈에 통제**하고 싶다 |
| 니즈 | ① 들어오는 주문을 2초 이내 확인 ② 테이블별 상황 한 화면에 ③ 메뉴·가격을 직접 즉시 수정 ④ 이상 주문 직권 삭제 ⑤ 매장 운영 종료 후 테이블 정리 |
| 페인 포인트 | POS 화면·주방 모니터·종이 메모가 분산, 알바 교대 시 인수인계 누락, 메뉴 수정에 외주 IT 호출 부담 |
| 디지털 친화도 | 중 — 간단한 사무용 SW 익숙, JWT·SSE 개념은 모름 (UI가 직관적이면 OK) |
| 보안 민감도 | 높음 — 매출 정보 외부 노출·타 매장 데이터 혼선은 절대 안 됨 |
| Voice Quote | "지금 4번 테이블에 뭐 들어갔는지 5초 안에 봐야 해. 한 번에 다 보이게 해줘." |

## P3 — 매장 알바생 "도윤"

| 항목 | 내용 |
|------|------|
| 이름 | 도윤 (20대 대학생, 파트타임) |
| 역할 | 점주 위임 권한으로 매장 운영 보조 — **점주와 동일 자격으로 로그인**(별도 권한 분리 없음, FR-A1 기준) |
| 사용 디바이스 | 매장 카운터 노트북 (브라우저) |
| 사용 환경 | 짧은 교대 시간(4~8시간), 점주 부재 시 단독 운영도 가능해야 함 |
| 핵심 동기 | 실수 없이 **빠르게 학습**해서 점주에게 핀잔 듣지 않고 싶다 |
| 니즈 | ① 처음 봐도 어디를 눌러야 하는지 명확 ② 신규 주문 알림이 시각적으로 강함 ③ 주문 상태 변경(대기→준비→완료)이 한 번에 ④ 테이블 이용 완료 처리 절차가 단순 ⑤ 잘못 누르면 확인 팝업 |
| 페인 포인트 | 신규 알바라 "이거 누르면 환불되는 거 아냐?" 같은 불안, 점주 자리 비울 때 의사결정 부담 |
| 디지털 친화도 | 상 — 스마트 디바이스 네이티브, 새 UI 학습 빠름 |
| 보안 민감도 | 낮음 (본인 입장) — 단 점주는 알바 실수로 매출/세션이 꼬이는 것을 매우 우려 |
| Voice Quote | "어제 막 들어왔는데 시킨 거 다 보여서 다행이에요. 빨간색 카드만 처리하면 되죠?" |
