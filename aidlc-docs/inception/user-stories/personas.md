# Personas (테이블오더 서비스)

총 3 페르소나 — Q1 답변에 따라 매장 고객 1 + 매장 운영자 2 (점주·알바생) 구성.

각 페르소나는 인구통계·역할·동기·니즈·페인 포인트·접속 컨텍스트·voice quote를 포함한다.

---

## P1 — 매장 고객 "진우"

| 항목 | 내용 |
|------|------|
| 이름 | 진우 (30대 직장인) |
| 역할 | 매장 테이블에 앉은 손님 — 주문 주체 |
| 사용 디바이스 | 매장 테이블에 비치된 **태블릿** (브라우저 풀스크린) |
| 사용 환경 | 매장 내부, 점심·저녁 피크 시간, 일행 1~3명 |
| 핵심 동기 | 직원을 부르지 않고 **즉시·정확하게** 메뉴를 보고 주문하고 싶다 |
| 니즈 | ① 별도 로그인 없이 바로 사용 ② 메뉴 사진·가격·설명 한눈에 ③ 장바구니로 일행 주문 모아서 확정 ④ 내 주문이 들어갔는지 확인 |
| 페인 포인트 | 종이 메뉴판은 사진 부족, 직원 호출은 대기, 모바일 앱 설치는 부담, 주문 누락·중복은 곤란 |
| 디지털 친화도 | 중~상 — 스마트폰 일상 사용, 처음 보는 UI도 30초면 익힘 |
| 보안 민감도 | 낮음 (결제 없음, 개인정보 입력 없음) — 단 매장 외부에 주문이 새지 않는다는 신뢰는 필요 |
| Voice Quote | "메뉴 보고 그냥 누르면 끝나는 거 아니야? 직원 부를 시간에 한 입 더 먹지." |

## P2 — 점주 "수민"

| 항목 | 내용 |
|------|------|
| 이름 | 수민 (40대 자영업자) |
| 역할 | 매장 소유주 — 매장 인증 자격 보유, **모든 관리 기능 권한** |
| 사용 디바이스 | 매장 후방 데스크탑 / 카운터 노트북 (브라우저) |
| 사용 환경 | 영업 중 카운터에서 실시간 모니터링 + 영업 외 시간 메뉴·계정 관리 |
| 핵심 동기 | 직원·인건비를 줄이면서 **매장 흐름을 한눈에 통제**하고 싶다 |
| 니즈 | ① 들어오는 주문을 2초 이내 확인 ② 테이블별 상황 한 화면에 ③ 메뉴·가격을 직접 즉시 수정 ④ 이상 주문 직권 삭제 ⑤ 매장 운영 종료 후 테이블 정리 ⑥ 신규 태블릿 초기 설정 |
| 페인 포인트 | POS 화면·주방 모니터·종이 메모가 분산, 알바 교대 시 인수인계 누락, 메뉴 수정에 외주 IT 호출 부담 |
| 디지털 친화도 | 중 — 간단한 사무용 SW 익숙, JWT·SSE 개념은 모름 (UI가 직관적이면 OK) |
| 보안 민감도 | 높음 — 매출 정보 외부 노출·타 매장 데이터 혼선은 절대 안 됨 |
| Voice Quote | "지금 4번 테이블에 뭐 들어갔는지 5초 안에 봐야 해. 한 번에 다 보이게 해줘." |

## P3 — 매장 알바생 "도윤"

| 항목 | 내용 |
|------|------|
| 이름 | 도윤 (20대 대학생, 파트타임) |
| 역할 | 점주 위임 권한으로 매장 운영 보조 — **점주와 동일 자격으로 로그인**(별도 권한 분리 없음, FR-A1 기준) |
| 사용 디바이스 | 매장 카운터 노트북 (브라우저) |
| 사용 환경 | 짧은 교대 시간(4~8시간), 점주 부재 시 단독 운영도 가능해야 함 |
| 핵심 동기 | 실수 없이 **빠르게 학습**해서 점주에게 핀잔 듣지 않고 싶다 |
| 니즈 | ① 처음 봐도 어디를 눌러야 하는지 명확 ② 신규 주문 알림이 시각적으로 강함 ③ 주문 상태 변경(대기→준비→완료)이 한 번에 ④ 테이블 이용 완료 처리 절차가 단순 ⑤ 잘못 누르면 확인 팝업 |
| 페인 포인트 | 신규 알바라 "이거 누르면 환불되는 거 아냐?" 같은 불안, 점주 자리 비울 때 의사결정 부담 |
| 디지털 친화도 | 상 — 스마트 디바이스 네이티브, 새 UI 학습 빠름 |
| 보안 민감도 | 낮음 (본인 입장) — 단 점주는 알바 실수로 매출/세션이 꼬이는 것을 매우 우려 |
| Voice Quote | "어제 막 들어왔는데 시킨 거 다 보여서 다행이에요. 빨간색 카드만 처리하면 되죠?" |

## 참고 — 페르소나 권한 모델 (MVP 가정)

| 페르소나 | 매장 인증 가능 | 메뉴 관리 | 주문 직권 삭제 | 테이블 세션 종료 | 비고 |
|----------|---------------|-----------|---------------|-----------------|------|
| P1 고객 진우 | ❌ (자동 로그인만) | ❌ | ❌ | ❌ | 태블릿 = 테이블 식별만 |
| P2 점주 수민 | ✅ | ✅ | ✅ | ✅ | 모든 관리 권한 |
| P3 알바 도윤 | ✅ | ✅ | ✅ | ✅ | **MVP 한정 — 점주와 동일 자격 공유**. 직원별 권한 분리는 Out of Scope. |

> **주의**: requirements.md 가 "매장 인증" 단일 모델(매장ID·사용자명·비밀번호)을 채택하므로, 점주·알바 모두 같은 자격으로 로그인한다. P3는 동일 자격을 "위임받아 쓰는" 사용자 컨텍스트 표현이며, 별도 권한 모델·역할 분리 기능은 본 MVP에 포함되지 않는다 (constraints.md "직원 관리 및 권한 설정" 제외).
