# 테이블오더 구현 예외사항

> **v2 갱신 (2026-06-19)**: BYOD/QR 전환 + 단방향 광고 슬롯 추가에 따라 "광고 기능 제외" 항목을 **부분 해제** + "푸시 알림" 항목 명확화 + "네이티브 앱" 명시. 자세한 결정 사유는 [aidlc-docs/inception/requirements/requirements.md §5](../aidlc-docs/inception/requirements/requirements.md) 참조.

## 제외 기능 (구현하지 않음)

### 1. 결제 관련
- 실제 결제 처리 (카드, 현금, 디지털 지갑)
- 결제 게이트웨이 연동 (PG사 연동)
- 영수증 발행 및 출력
- 환불 처리
- 포인트/쿠폰 시스템

### 2. 인증 및 보안
- 복잡한 사용자 인증 (OAuth, SNS 로그인)
- 다단계 인증 (2FA, OTP)

### 3. 파일 및 컨텐츠 관리
- 이미지 리사이징/최적화
- 컨텐츠 관리 시스템
- ~~광고 기능 등은 포함하지 않음~~ → **v2 부분 해제**: 단방향 광고 슬롯(모두의주차장 배너 1~2개, 시드 데이터로 정적 제공)은 **포함**. 다음은 여전히 **제외**: 광고주 등록 UI·노출 통계·정산·매장별 광고 on/off·교체

### 4. 알림 시스템
- 푸시 알림 (모바일 네이티브, Web Push 포함) — **SSE 화면 알림으로 대체**
- SMS 알림
- 이메일 발송
- 소리/진동 알림

### 5. 주방 기능
- 주문 내역 주방 전달
- 주방 식재료 재고 관리

### 6. 고급 기능
- 데이터 분석 및 대시보드
- 매출 리포트 생성
- 재고 관리 시스템
- 직원 관리 및 권한 설정
- 예약 시스템
- 고객 리뷰 시스템
- 다국어 기능

### 7. 외부 연동
- 배달 플랫폼 연동
- POS 시스템 연동
- 소셜 미디어 공유
- 지도 API
- 번역 API

### 8. 클라이언트 형태 (v2 신규)
- 네이티브 앱 (iOS/Android) — **모바일 웹/PWA만 지원**
